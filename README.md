## 2. 프로젝트 개요

### 프로젝트명
**CALL:ACT** - 카드사 콜센터 상담원을 위한 AI 상담 업무 지원 서비스

### 프로젝트 기간
2025년 1월 ~ 2025년 2월

### 프로젝트 목표
LLM과 RAG 기술을 활용하여 콜센터 상담원의 업무 효율성을 향상시키는 AI 기반 상담 지원 플랫폼 개발

### 주요 기능

| 기능 | 설명 |
|------|------|
| **실시간 상담 지원** | WebSocket 기반 STT, RAG 검색, AI 응답 추천 |
| **교육 시뮬레이션** | 가상 고객 대화 연습, TTS 피드백, 성과 평가 |
| **사후 처리 (ACW)** | 상담 내용 자동 요약, AI 피드백 생성 |
| **고객/상담사 관리** | 고객 정보 조회, 상담 이력 관리 |
| **공지사항/FAQ** | 공지사항 관리, 자주 찾는 문의 제공 |

## 4. 문제 정의

### 콜센터 상담원이 겪는 문제점

#### 1. 정보 검색의 비효율성
- 복잡한 금융 상품 정보를 실시간으로 검색하기 어려움
- 키워드 기반 검색의 한계로 원하는 정보를 찾는 데 시간 소요
- 상담 중 고객 대기 시간 증가

#### 2. 신입 상담원의 높은 학습 곡선
- 다양한 상품과 서비스에 대한 지식 습득에 오랜 시간 필요
- 실제 상담 경험 부족으로 인한 응대 품질 저하
- 체계적인 교육 시스템 부재

#### 3. 사후 처리 업무 부담
- 상담 종료 후 내용 정리 및 요약에 많은 시간 소요
- 수작업으로 인한 기록 누락 및 오류 발생
- 상담 품질 분석의 어려움

## 5. 문제 해결 전략

### 해결 방안

| 문제 | 해결 전략 | 적용 기술 |
|------|----------|----------|
| 정보 검색 비효율 | RAG 기반 의미 검색 | pgvector, LlamaIndex, GPT-4 |
| 실시간 응대 지연 | 실시간 STT + AI 추천 | Whisper, WebSocket |
| 신입 교육 부족 | AI 시뮬레이션 교육 | TTS, LLM 기반 가상 고객 |
| 사후 처리 부담 | 자동 요약 및 피드백 | GPT-4 기반 요약/분석 |

### 핵심 기술 적용

#### RAG (Retrieval-Augmented Generation)
```
사용자 질문 → 키워드 추출 → 벡터 검색 → 재랭킹 → 응답 생성
```
- 형태소 분석을 통한 금융 키워드 추출
- pgvector를 활용한 의미 기반 문서 검색
- LLM 기반 검색 결과 재정렬 및 응답 생성

#### 실시간 음성 처리
```
음성 입력 → STT (Whisper) → 화자 분리 → 텍스트 처리 → RAG 검색
```
- WebSocket 기반 실시간 통신
- Faster-Whisper를 활용한 고속 음성 인식
- 화자 분리(Diarization)를 통한 상담원/고객 구분

## 6. 수집 데이터 및 활용 목적

### 수집 데이터

| 데이터 유형 | 내용 | 활용 목적 |
|------------|------|----------|
| **카드 상품 정보** | 신용카드 상품명, 혜택, 연회비 등 | RAG 검색 대상 |
| **서비스 가이드** | 카드 발급, 분실 신고, 결제 방법 등 | 상담 응답 생성 |
| **FAQ 데이터** | 자주 묻는 질문과 답변 | 빠른 응답 제공 |
| **상담 시나리오** | 다양한 상담 상황 시나리오 | 교육 시뮬레이션 |

### 데이터 전처리

1. **텍스트 정제**: 불필요한 특수문자 제거, 정규화
2. **청킹(Chunking)**: 문서를 적절한 크기로 분할
3. **임베딩**: OpenAI Embedding 모델을 통한 벡터화
4. **인덱싱**: pgvector에 벡터 데이터 저장

## 8. 성과 및 검증

### 주요 성과

| 지표 | 내용 |
|------|------|
| **검색 정확도** | RAG 파이프라인을 통한 맥락 기반 정확한 정보 제공 |
| **응답 속도** | WebSocket 기반 실시간 STT 및 응답 추천 |
| **교육 효과** | AI 시뮬레이션을 통한 체계적인 상담원 교육 |
| **업무 효율** | 자동 요약 기능으로 사후 처리 시간 단축 |

### 구현 기능 검증

- **실시간 상담**: WebSocket 연결 안정성, STT 정확도 테스트
- **RAG 검색**: 다양한 질문에 대한 검색 결과 품질 평가
- **교육 시뮬레이션**: 가상 고객 응답의 자연스러움 검증
- **사후 처리**: 요약 정확도 및 피드백 유용성 평가

## 9. 트러블 슈팅

### 1. WebSocket 연결 불안정
**문제**: 장시간 상담 시 WebSocket 연결이 끊기는 현상
```
해결:
- Heartbeat 메커니즘 구현
- 자동 재연결 로직 추가
- 연결 상태 모니터링 강화
```

### 2. STT 지연 문제
**문제**: 음성 인식 결과가 늦게 반환되어 실시간성 저하
```
해결:
- Faster-Whisper 도입으로 처리 속도 개선
- 청크 단위 처리로 점진적 결과 반환
- 비동기 처리 최적화
```

### 3. RAG 검색 품질
**문제**: 키워드만으로는 의도에 맞는 문서 검색 어려움
```
해결:
- 형태소 분석을 통한 키워드 추출 고도화
- 재랭킹(Reranking) 단계 추가
- 임베딩 캐시를 통한 성능 최적화
```

### 4. 대용량 PDF 처리
**문제**: 금융 문서 PDF 파싱 시 메모리 부족
```
해결:
- 스트리밍 방식 PDF 처리
- 청킹 크기 최적화
- 배치 처리 도입
```
